Modular\Traits\backpropEventInfo (   $event)

Return the event info for the event, override to provide results that can't be declared statically in config. What to store for each event is declared as config.backprop_events on each model, with event as key and a tag as the value. The tag can be a method name to call on the model, a field to copy the value from, a property (not field) on the model or if none of those are available a verbatim value.

Parameters
string$eventname of event, e.g. 'changing'
Returns
mixed results from dereferencing the value from the event

backprop
Initial value:
{
protected $backprop

Notify related models (e.g. owning Page via HasPage or Related pages via RelatedPages ) so it shows as modified in CMS when this model changes.

Care should be taken that this doesn't happen as part or after a publish, otherwise if the model is published then the owner will be marked as dirty and so will always appear as being 'modified' to the CMS.

Parameters
string$eventgets passed to related models, could be an 'event name' e.g. 'published' or an originating method name e.g. 'onAfterWrite'.
